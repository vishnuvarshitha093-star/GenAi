Functional Requirements Document

Scope

Workplace Access & Resource Request Management covering system access, equipment, and facilities.

Actors

  • User
  • Approver
  • Admin

Lifecycle

Submitted → Under Review → Approved/Rejected → Fulfilled → Closed.

Business Rules

  • Edit only when Submitted.
  • Fulfill only when Approved.
  • Close only when Fulfilled.
  • Closed read-only.
  • SLA due date auto-calculated by category.
  • Audit logging mandatory for all CUD and status transitions.

Non-Functional

  • JWT auth with RBAC.
  • PostgreSQL + SQLAlchemy + Alembic.
  • Export to PDF/CSV/Excel.

  3. Schema validation against OpenAPI and business rules.
  4. Code scaffolding via AI-assisted templates.
  5. Test case generation from state model.
  6. API contract linting versus openapi.yaml.
  7. HTML doc generation from markdown specs.
  8. Code quality checker AI agent in Streamlit + LangChain.

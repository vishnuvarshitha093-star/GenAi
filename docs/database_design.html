Database Design

Logical Model

  • User(1) to ResourceRequest(M) by requested_by_id.
  • User(1) to ResourceRequest(M) by approver_id.
  • ResourceRequest(1) to AuditLog(M) via entity_id + entity_type.

Physical Model

PostgreSQL tables: users, resource_requests, audit_logs. Enums for role and status.

ER Diagram Explanation

Users own requests; approvers act on workflow; audit logs capture immutable action history for compliance.
